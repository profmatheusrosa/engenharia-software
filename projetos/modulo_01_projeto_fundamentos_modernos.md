# Projeto - Módulo 1: Setup e Cultura

## Título do Projeto: Início da Jornada Ágil

## Descrição
Neste primeiro módulo, o projeto é focado na estruturação do ambiente de trabalho e na adoção de práticas ágeis. Você simulará o início de um "Greenfield Project" (projeto do zero).

## Cenário
Você foi contratado como Engenheiro de Software Líder para uma startup que vai criar um sistema de gestão de tarefas (To-Do List Enterprise). Antes de escrever código, você precisa estabelecer o processo.

## Requisitos e Etapas

1. **Configuração do Repositório:**
   - Crie um repositório no GitHub para o seu projeto pessoal do curso (se já não tiver criado).
   - Garanta que ele tenha um `.gitignore` adequado para Node.js/Java/Python (sua linguagem de preferência).
   - Adicione um `README.md` descrevendo o projeto "To-Do List Enterprise".

2. **Gestão de Backlog:**
   - Usando a aba "Projects" do GitHub (ou Trello), crie o backlog inicial do produto.
   - Escreva pelo menos 5 User Stories iniciais para o sistema de To-Do List (ex: "Como usuário, quero adicionar uma tarefa para não esquecer meus compromissos").

3. **Definição de Processo:**
   - Crie um arquivo `PROCESSO.md` na raiz do seu repositório.
   - Descreva como será o fluxo de trabalho da equipe (fictícia). Defina:
     - Duração da Sprint (ex: 2 semanas).
     - Dia e horário da Daily Scrum.
     - Definição de "Done" (DoD) para uma tarefa ser considerada pronta (ex: Código, Testes Passando, Code Review Aprovado).

4. **Entrega:**
   - O link do repositório contendo o `README.md` e o `PROCESSO.md`, além do link para o Board do projeto.
