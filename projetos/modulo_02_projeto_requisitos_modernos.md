# Projeto - Módulo 2: Product Discovery

## Título do Projeto: Mapeando o Produto

## Descrição
Agora que você tem um backlog inicial, é hora de organizar a visão do produto para garantir que estamos construindo a coisa certa.

## Requisitos e Etapas

1. **Story Mapping:**
   - Utilize uma ferramenta online (Miro, Mural, ou o próprio draw.io) para criar um Story Map do "To-Do List Enterprise".
   - **Backbone (Espinha Dorsal):** Identifique as grandes etapas da jornada do usuário (ex: Cadastro -> Criação de Listas -> Gestão de Tarefas -> Relatórios).
   - **Body:** Quebre essas etapas em user stories menores.
   - **Slicing:** Trace uma linha definindo o que entra no MVP (Minimum Viable Product/Release 1) e o que fica para depois.

2. **Modelo de Domínio:**
   - Crie um diagrama simples representando as principais entidades do seu sistema.
   - Identifique: `User`, `Task`, `List`, `Tag` (ou `Category`).
   - Mostre como elas se relacionam (ex: "Um User pode ter várias Lists", "Uma List tem várias Tasks").
   - Salve a imagem na pasta `docs/` ou `imagens/` do seu repositório pessoal.

3. **Atualização do README:**
   - Adicione uma seção "Visão do Produto" no seu README.md e inclua o link para a imagem do Modelo de Domínio e, se possível, um link ou print do Story Map.
