# Projeto - Módulo 6: Garantindo a Qualidade

## Título do Projeto: Testando o To-Do List

## Descrição
Vamos adicionar testes automatizados ao nosso projeto principal.

## Requisitos e Etapas

1.  **Testes Unitários:**
    - Crie testes unitários para a entidade `Task`. Ex: Garantir que não se pode criar uma task sem título ou com título vazio.
    - Se usou Factory no módulo anterior, teste a Factory.

2.  **Testes de Integração (Opcional):**
    - Se tiver conhecimento, crie um teste que simula o Service salvando no Repository (em memória) e depois recuperando para ver se os dados batem.

3.  **Configuração de CI Local (Pre-commit Hook):**
    - (Avançado) Instale o `husky` (se estiver usando Node) para rodar `npm test` antes de cada commit. Isso garante que nunca será commitado código quebrado.

## Entrega
Print do terminal rodando os testes com sucesso (tudo verde).
