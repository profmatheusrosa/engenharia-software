# Glossário de Engenharia de Software Moderna

## A
- **Agile (Ágil):** Metodologia de desenvolvimento de software focada em entrega iterativa, colaboração e resposta a mudanças.
- **API (Application Programming Interface):** Conjunto de definições que permite que softwares se comuniquem.

## B
- **Backlog:** Lista priorizada de itens (funcionalidades, correções, melhorias) a serem feitos no produto.
- **Bounded Context:** Parte central do DDD. Delimita onde um determinado modelo de domínio se aplica e faz sentido.

## C
- **CI/CD:** Integração Contínua e Entrega/Implantação Contínua. Prática de automatizar o build, teste e deploy.
- **Clean Architecture:** Padrão arquitetural que isola as regras de negócio de detalhes externos como UI e Banco de Dados.

## D
- **DDD (Domain-Driven Design):** Abordagem de design de software focada na modelagem do domínio do problema.
- **DevOps:** Cultura e práticas que unem desenvolvimento e operações para encurtar o ciclo de vida de desenvolvimento.

## E
- **E2E (End-to-End):** Testes que verificam o fluxo completo da aplicação, simulando o usuário final.

## G
- **Git Flow:** Modelo de ramificação (branching) do Git projetado para gerenciar releases de forma estrita.

## I
- **INVEST:** Acrônimo para boas User Stories: Independent, Negotiable, Valuable, Estimable, Small, Testable.

## K
- **Kanban:** Método visual para gerenciar fluxo de trabalho, focado em limitar o trabalho em progresso (WIP).

## M
- **Microserviços:** Estilo arquitetural onde a aplicação é composta por pequenos serviços independentes.
- **Mock:** Objeto simulado usado em testes para substituir dependências reais.
- **Monólito:** Aplicação onde todo o código é empacotado e implantado como uma unidade única.

## P
- **PO (Product Owner):** Papel no Scrum responsável por maximizar o valor do produto e gerenciar o Backlog.
- **Pull Request (PR):** Solicitação para mesclar alterações de uma branch em outra, geralmente envolvendo revisão de código.

## R
- **Refactoring:** Processo de reestruturar o código sem mudar seu comportamento externo, para melhorar legibilidade e estrutura.

## S
- **Scrum:** Framework ágil para gestão de projetos complexos.
- **SOLID:** Cinco princípios de design orientado a objetos para criar software manutenível.

## T
- **TDD (Test Driven Development):** Desenvolvimento guiado por testes. Red -> Green -> Refactor.
- **User Story:** Descrição curta e simples de uma funcionalidade sob a perspectiva do usuário.
